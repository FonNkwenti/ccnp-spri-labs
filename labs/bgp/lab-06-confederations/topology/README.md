# Lab 06: BGP Confederations — Topology

## Summary

This topology spans **three visible autonomous systems** (AS 65001, AS 65100, AS 65002)
connected in a linear arrangement. AS 65100 is a confederation divided into two sub-ASes
internally — sub-AS 65101 (East PEs) and sub-AS 65102 (Core + West PE). It contains
**6 routers** and **7 physical links**.

```
AS 65001 (R1 CE)
    |
    |-- L1 eBGP primary   --> R2 PE East-1 (Sub-AS 65101) --|
    |-- L2 eBGP backup    --> R3 PE East-2 (Sub-AS 65101) --|
                                                            |-- L6 iBGP R2<->R3
                           L3 confed-eBGP (65101->65102) --> R4 SP Core (Sub-AS 65102)
                           L4 confed-eBGP (65101->65102) --> R4
                                                             |-- L5 iBGP R4<->R5 --> R5 PE West (Sub-AS 65102)
                                                                                      |-- L7 eBGP --> R6 (AS 65002)
```

### Layout Description

| Zone | Devices | Position |
|------|---------|----------|
| AS 65001 | R1 (CE) | Left |
| Confederation 65100 — Sub-AS 65101 | R2 (PE East-1), R3 (PE East-2) | Center-left |
| Confederation 65100 — Sub-AS 65102 | R4 (SP Core), R5 (PE West) | Center-right |
| AS 65002 | R6 (External SP Peer) | Right |

OSPF area 0 runs across the confederation core on links L3, L4, L5, L6 and all SP loopbacks.
R1 and R6 have NO OSPF — they are external to the confederation.
R5 is CSR1000v (IOS-XE 17.3); all others are IOSv (IOS 15.9).

## EVE-NG Import Instructions

1. Open the EVE-NG web UI and log in.
2. Navigate to **File > Import** (or use the Labs menu > Import).
3. Select the `.unl` file for this lab (generated by `setup_lab.py`).
4. After import, the lab will appear in your lab list.
5. Open the lab — all nodes and connections are pre-wired per this topology.

## Starting the Lab

1. Open the lab in EVE-NG.
2. Select all nodes (Ctrl+A) and click **Start** (or right-click > Start).
3. Wait approximately 2-3 minutes for all IOSv nodes to boot and 4-5 minutes for R5
   (CSR1000v) to reach a usable prompt.
4. Run the setup script to push the pre-configured base configuration (IP + OSPF only):

   ```bash
   python labs/bgp/lab-06-confederations/setup_lab.py --host <eve-ng-ip>
   ```

5. Console into any device via:

   ```bash
   telnet <eve-ng-ip> <console-port>
   ```

   Console port numbers are shown in the EVE-NG UI node properties panel.

## Exporting the Lab

To export the lab as a `.unl` archive for sharing or backup:

1. In the EVE-NG web UI, right-click the lab in the lab list.
2. Select **Export** and save the `.unl` file.
3. Alternatively, from the EVE-NG shell, the lab files are located at:

   ```
   /opt/unetlab/labs/<your-folder>/<lab-name>.unl
   ```
