# Topology — MPLS Lab 01: LSP Verification with MPLS OAM

Four-router IOSv SP core in a diamond topology: PE1 top-left, P1 top-right, P2 bottom-left, PE2 bottom-right, with a P1↔P2 cross (L4) that creates two equal-cost paths between PE1 and PE2. Five core links total (L2–L6). No CE devices — this lab works entirely within the core.

## EVE-NG Import Instructions

1. Log in to the EVE-NG web UI.
2. Navigate to the folder where you want to place the lab (e.g., `ccnp-spri/mpls/`).
3. Click **File > Import** and select `lab-01-lsp-verification-and-troubleshooting.unl`.
4. The lab appears in the folder. Open it to verify the topology matches the diagram below.

> The `.unl` file is created manually in EVE-NG after building the topology — it is not auto-generated by the build scripts. `topology.drawio` is the visual reference; use it when creating or re-creating the lab in EVE-NG.

## Starting the Lab

1. In the EVE-NG web UI, open the lab and click **Start all nodes**.
2. Wait approximately 60 seconds for IOSv to boot.
3. Console ports are assigned dynamically by EVE-NG — check the UI or use the REST API.
4. Run `setup_lab.py --host <eve-ng-ip>` to push initial configs to all four devices.

## Exporting the Lab

After making topology changes in EVE-NG (adding nodes, changing links):

1. In the EVE-NG web UI, click **File > Export**.
2. Save the `.unl` file alongside this README in the `topology/` folder.
3. Commit the updated `.unl` to git to preserve the topology snapshot.
